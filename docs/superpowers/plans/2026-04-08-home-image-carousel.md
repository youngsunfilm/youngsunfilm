# Home Image Carousel Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an auto-advancing horizontal image strip to the home page, sourced from a new `carouselImages` field on the `home` Sanity singleton, rendered between the intro header and the showcase projects list.

**Architecture:** Server/client split. A new Server Component (`HomeCarousel.tsx`) reads the typed Sanity data and renders a scroll-snap list of `next/image` elements wrapped in a small Client Component (`HomeCarouselAutoplay.tsx`) that owns a scroll ref, a `setInterval`, and pause-on-hover behavior. Visual Editing data attributes reuse the existing `createDataAttribute` pattern from `HomePage.tsx`.

**Tech Stack:** Next.js 16.2.2 (App Router, Turbopack, React 19.2), Sanity 5.19 with `next-sanity` + TypeGen, Tailwind 3.4, `@sanity/image-url`.

**Test strategy:** This repo has no test runner (no Vitest/Jest/Playwright in `package.json`) and the design spec explicitly declines to add one for this feature. Verification gates instead use `npm run type-check` (which runs `sanity typegen generate` + `tsc --noEmit`) plus a manual end-to-end check described in the final task. Every task that changes code ends with a type-check run before committing.

- [ ] **Step 2: Add `carouselImages` to the projection**

Replace the entire `homePageQuery` export with:

```ts
export const homePageQuery = defineQuery(`
  *[_type == "home"][0]{
    _id,
    _type,
    overview,
    carouselImages[]{
      ...,
    },
    showcaseProjects[]{
      _key,
      ...@->{
        _id,
        _type,
        coverImage,
        overview,
        "slug": slug.current,
        tags,
        title,
      }
    },
    title,
  }
`)
```

Rationale for the `carouselImages[]{...,}` spread rather than an explicit field list:
- `@sanity/image-url`'s `urlForImage` helper in this repo expects Sanity's native `Image` type (which includes `_type: 'image'`, `asset`, `hotspot`, `crop`). An explicit projection that forgets `_type` produces a type mismatch.
- The spread (`...`) selects all fields of the image object — including the custom `alt` field we just added to the schema — and keeps the TypeGen'd type structurally compatible with `Image`.
- The asset is intentionally not dereferenced (`asset->`) because the image URL builder only needs the asset reference, not the full asset document.

## Task 3: Create the `HomeCarouselAutoplay` client component

**Files:**
- Create: `components/HomeCarouselAutoplay.tsx`

This is the smallest client surface: it receives already-rendered `children` and wraps them with a `ref`, `interval`, and pause behavior. It has no Sanity imports and no image logic.

- [ ] **Step 1: Create the file with the full implementation**

Create `components/HomeCarouselAutoplay.tsx`:

```tsx
'use client'

import {useEffect, useRef} from 'react'

export interface HomeCarouselAutoplayProps {
  children: React.ReactNode
  /** Number of carousel items. Used to skip autoplay when there is ≤1. */
  itemCount: number
  /** Autoplay tick interval in milliseconds. */
  intervalMs?: number
}

/**
 * Wraps a horizontally scrollable list and steps it forward on an interval.
 *
 * - Pauses while the pointer is hovering the container.
 * - Pauses briefly while the user is actively scrolling and resumes after idle.
 * - Respects `prefers-reduced-motion`.
 * - Renders only a div; the actual `<ul>` and `<li>` markup is passed in as children.
 */
export function HomeCarouselAutoplay({
  children,
  itemCount,
  intervalMs = 4000,
}: HomeCarouselAutoplayProps) {
  const scrollerRef = useRef<HTMLDivElement>(null)

  useEffect(() => {
    const scroller = scrollerRef.current
    if (!scroller) return
    if (itemCount <= 1) return

    const prefersReducedMotion =
      typeof window !== 'undefined' &&
      window.matchMedia('(prefers-reduced-motion: reduce)').matches
    if (prefersReducedMotion) return

    let paused = false
    let userScrollIdleTimer: ReturnType<typeof setTimeout> | null = null

    const step = () => {
      if (paused) return
      const firstChild = scroller.firstElementChild as HTMLElement | null
      if (!firstChild) return
      const gap = parseFloat(getComputedStyle(scroller).columnGap || '0') || 0
      const stepPx = firstChild.offsetWidth + gap
      const atEnd = scroller.scrollLeft + scroller.clientWidth >= scroller.scrollWidth - 1
      if (atEnd) {
        scroller.scrollTo({left: 0, behavior: 'auto'})
      } else {
        scroller.scrollBy({left: stepPx, behavior: 'smooth'})
      }
    }

    const tick = setInterval(step, intervalMs)

    const onPointerEnter = () => {
      paused = true
    }
    const onPointerLeave = () => {
      paused = false
    }
    const onScroll = () => {
      paused = true
      if (userScrollIdleTimer) clearTimeout(userScrollIdleTimer)
      userScrollIdleTimer = setTimeout(() => {
        paused = false
      }, 1000)
    }

    scroller.addEventListener('pointerenter', onPointerEnter)
    scroller.addEventListener('pointerleave', onPointerLeave)
    scroller.addEventListener('scroll', onScroll, {passive: true})

    return () => {
      clearInterval(tick)
      if (userScrollIdleTimer) clearTimeout(userScrollIdleTimer)
      scroller.removeEventListener('pointerenter', onPointerEnter)
      scroller.removeEventListener('pointerleave', onPointerLeave)
      scroller.removeEventListener('scroll', onScroll)
    }
  }, [itemCount, intervalMs])

  return (
    <div
      ref={scrollerRef}
      className="flex snap-x snap-mandatory gap-4 overflow-x-auto scroll-smooth [scrollbar-width:none] [&::-webkit-scrollbar]:hidden"
    >
      {children}
    </div>
  )
}
```

Notes:
- `paused` is a closure variable, not React state — we don't need re-renders, we just need the interval to read a mutable flag.
- `columnGap` from `getComputedStyle` resolves the Tailwind `gap-4` into pixels at runtime. This handles viewport resizes without needing a listener because we re-measure on every tick.
- The scrollbar-hiding classes use Tailwind's arbitrary-variant syntax (`[&::-webkit-scrollbar]:hidden`) which works in Tailwind 3.4 (in this repo).
- The component intentionally holds no state about `itemCount` beyond the gating check, so it cannot drift out of sync with the server-rendered children.

## Task 4: Create the `HomeCarousel` server component

**Files:**
- Create: `components/HomeCarousel.tsx`

- [ ] **Step 1: Create the file with the full implementation**

Create `components/HomeCarousel.tsx`:

```tsx
import {HomeCarouselAutoplay} from '@/components/HomeCarouselAutoplay'
import type {HomePageQueryResult} from '@/sanity.types'
import {studioUrl} from '@/sanity/lib/api'
import {urlForImage} from '@/sanity/lib/utils'
import {createDataAttribute} from 'next-sanity'
import Image from 'next/image'

type CarouselImages = NonNullable<HomePageQueryResult>['carouselImages']

export interface HomeCarouselProps {
  images: CarouselImages | null | undefined
  id: string | null
  type: string | null
}

/**
 * Horizontal auto-advancing image strip rendered on the home page between the
 * intro header and the showcase projects list. Server Component — all image
 * URL resolution and <Image> rendering happens here; only the interval/ref
 * behavior ships to the browser via HomeCarouselAutoplay.
 */
export function HomeCarousel({images, id, type}: HomeCarouselProps) {
  if (!images || images.length === 0) return null

  const resolved = images
    .map((image) => {
      if (!image) return null
      const url = urlForImage(image)?.width(1600).height(1066).url()
      if (!url) return null
      return {
        key: image._key,
        url,
        alt: image.alt ?? '',
      }
    })
    .filter((item): item is {key: string; url: string; alt: string} => item !== null)

  if (resolved.length === 0) return null

  const dataAttribute =
    id && type
      ? createDataAttribute({baseUrl: studioUrl, id, type})
      : null

  return (
    <section
      aria-label="Featured images"
      data-sanity={dataAttribute?.(['carouselImages'])}
    >
      <HomeCarouselAutoplay itemCount={resolved.length}>
        {resolved.map((item, index) => (
          <div
            key={item.key}
            className="relative aspect-[3/2] w-[80%] shrink-0 snap-start overflow-hidden rounded-md md:w-[45%] lg:w-[32%]"
          >
            <Image
              src={item.url}
              alt={item.alt}
              fill
              sizes="(min-width: 1024px) 32vw, (min-width: 768px) 45vw, 80vw"
              className="object-cover"
              priority={index === 0}
            />
          </div>
        ))}
      </HomeCarouselAutoplay>
    </section>
  )
}
```

Notes:
- `urlForImage(...).width(1600).height(1066)` pre-bakes a 3:2 crop at roughly the target display size. `next/image`'s own optimizer will downsize further for smaller viewports via `sizes`.
- `resolved` is built with a `.map().filter()` narrowing pattern so TypeScript knows the final array has no `null` entries.
- The `section` gets `aria-label` — without it, a region with no visible heading is invisible to landmark navigation.
- The `data-sanity` attribute is only rendered when we have both `id` and `type`, mirroring the guard pattern in `components/HomePage.tsx`.
